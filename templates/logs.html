Comment configurer l'envoi des logs depuis le routeur ?
MÉTHODE 1 Via SSH / Console (CLI)

Connectez-vous en SSH au routeur, puis entrez les commandes suivantes en mode système :

# Activer le centre d'information
info-center enable

# Déclarer le serveur syslog (remplacez l'IP)
info-center loghost {{ request.host.split(':')[0] }} facility local6

# Activer l'envoi sur le canal syslog
info-center loghost source GigabitEthernet0/0/0

# Vérification
display info-center

Port UDP standard : 514 — mappé automatiquement vers le récepteur interne. Si votre routeur propose un champ port, entrez 514.

MÉTHODE 2 Via l'interface Web eWeb (sans SSH)

Accédez à l'interface web du routeur (généralement http://192.168.x.x) puis suivez le chemin :

1
Accès au menu principal
Menu Diagnostic → Info-center
Ou selon la version firmware : System → Log Management → Loghost
2
Activer l'info-center
Cochez/activez "Info-center Enable" si ce n'est pas déjà fait.
3
Ajouter un loghost
Cliquez Ajouter et renseignez :
Log host IP : {{ request.host.split(':')[0] }}
L'AR651W n'expose pas de champ "port" dans eWeb — il utilise toujours le port standard UDP 514. HuwaControl redirige automatiquement ce port vers son récepteur interne.
4
Sauvegarder et tester
Cliquez Appliquer / OK. Les logs devraient apparaître dans cette page sous quelques secondes. Si rien n'apparaît, vérifiez que le port UDP {{ syslog_port }} est accessible (pas de firewall entre le routeur et ce serveur).
